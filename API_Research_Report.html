1. Federal-Level Data (Congress)

Congress.gov API FREE IN USE

What it covers: Members of Congress (House & Senate), bills, amendments, votes, nominations, treaties, committee reports, and summaries. This is now the official replacement for the retired ProPublica Congress API.

Data includes: Member biographical info, party, state, sponsored/cosponsored legislation, voting records (including individual roll-call votes), bill text and status.

Access: Free API key via Data.gov. Returns JSON or XML.

Limits: Standard rate limits (1,000 requests/hour typical for free keys).

Gaps: No photos. No local/state officials. Federal only.

GitHub Documentation | API Portal

United States Congress Images FREE / PUBLIC DOMAIN IN USE

What it covers: Headshot photos of all current and many past members of Congress.

How it works: Photos hosted at predictable URLs by Bioguide ID:
https://unitedstates.github.io/images/congress/450x550/{bioguide_id}.jpg

Source: Government Printing Office (public domain, CC0 dedication).

OpenFEC API (Campaign Finance) FREE

What it covers: Campaign contributions, expenditures, donor data, PAC information, and candidate financial filings for all federal races.

Access: Free API key via api.open.fec.gov.

Useful for: Showing who funds candidates, financial transparency data.

1b. Profile Content — Bios, Issue Positions & Committees

The fields a rep profile shows beyond name/party/votes. Congress.gov gives us a factual bio + congressional experience automatically (the app derives both from the member terms). The harder, editorial pieces — issue positions / stances and richer narrative bios — have no single complete free source; the options below fill that gap.

Vote Smart API (Project Vote Smart) PARTIAL COVERAGE PLANNED (budget permitting)

What it covers: The strongest free source for issue positions. Non-partisan; provides biographies, issue positions via the Political Courage Test (PCT), voting records, interest-group ratings, campaign finance, and public statements — for President, Congress, Governor, and state legislature.

Why it matters here: The one profile field Congress.gov can't give us. PCT positions are non-partisan and candidate-submitted.

Caveat: Coverage is incomplete — many incumbents decline the PCT, so positions exist for some members but not all. Use as a supplement (fill where available), not a complete feed. API access for a public-facing product requires a paid subscription — quoted June 2026 at $4,850/year for the Public-Facing Platform License (developer/testing tiers $350/mo, $1,000/3mo, $1,900/6mo, non-public only). Deferred until post-launch budget allows; we ship neutral, sourced issue summaries derived from legislative records in the meantime.

Vote Smart API | About the PCT

Wikidata REST API + Wikipedia REST API FREE / CC0

What it covers: Free narrative bios + structured facts (education, prior offices, dates) for essentially every member of Congress. Technique: fuzzy-match the name to a Wikipedia article, grab the Wikidata QID, read structured statements; the Wikipedia REST API adds a plain-text summary + thumbnail.

Why it matters here: A richer "About" than the auto one-liner, plus pre-Congress facts the Congress.gov API lacks. Caveat: community-edited — verify before publishing on a civic platform.

Wikidata REST API

Biographical Directory of the U.S. Congress (bioguide) FREE / PUBLIC DOMAIN

What it covers: Official short biographies for current + historical members, keyed by the same Bioguide ID we already use for photos and the Congress.gov API. Authoritative and neutral.

bioguide.congress.gov

Congress.gov — Committee Assignments + Sponsored-bill Subjects FREE

What it covers: Two things we have access to but aren't fully using: (1) committee / subcommittee assignments per member (the profile's Committees list, currently empty, can be auto-filled like bio/experience now are); (2) the policy areas / legislative subjects of a member's sponsored bills — a complete-coverage, derivable signal of "what they actually work on" that can seed the Issues tab even where no curated PCT positions exist.

2. State-Level Data (Governors, State Legislators)

Open States / Plural API FREE (with limits) IN USE

What it covers: State legislators for all 50 states + DC + Puerto Rico. Includes bills, votes, committees, and legislator profiles.

Data includes: Legislator names, party, district, contact info, committee assignments, bill sponsorship, vote records.

Access: Free API key from open.pluralpolicy.com. Also offers bulk data downloads.

Limits: Free tier designed for most users. Higher usage may require contact with Plural.

Gaps: Limited municipal coverage. Does not cover sheriffs, judges, DAs, or school board members.

API v3 Documentation

State Judiciary — State Supreme & Appellate Courts

CourtListener / Free Law Project FREE IN USE (opinions) PLANNED (judges)

What it covers: A People/Judges database of ~16,000+ federal and state judges — biographical and educational background, courts and positions held, political affiliation, ABA ratings, and authored opinions — plus full case-law/opinions. Strongest for state supreme and intermediate appellate courts; thinner for trial / circuit / county judges.

Why it matters here: The app already uses CourtListener for state supreme-court opinions (see state_live.py, STATE_SUPREME_COURT_CODES + COURTLISTENER_TOKEN). The People API is the planned source to populate the judiciary.supreme_court roster for the big states, mirroring Florida's existing judiciary block.

Access: Free, non-profit (Free Law Project, 501(c)(3)). REST API v4 + bulk data. Caveat: as of May 2026, full API access is tied to a Free Law Project membership; default/free rates remain for light use.

Gaps: No trial/county judges or DAs — that tier remains a Ballotpedia (paid) / curated gap, same as local officials.

3. Local-Level Data (The Hard Part)

Ballotpedia API PAID (contact for pricing)

What it covers: The most comprehensive source for local political data. Covers ~15,000 political offices including state legislators, mayors, city council, sheriffs, judges, DAs, and school board members in select states.

Data includes: Names, party, incumbency, biographical info, contact info, campaign websites, district maps, election dates.

Down-ballot coverage (2026): AZ, AR, CA, DE, FL, GA, HI, ID, IL, IN, IA, KY, ME, MI, MN, MT, NV, NM (expanding).

Cost: Commercial pricing — must contact data@ballotpedia.org for rate card. Likely $200-1000+/month depending on usage.

Why this matters: This is the ONLY comprehensive source for local officials like sheriffs, judges, and DAs — exactly what your app needs.

Developer Docs | Data Store

Google Civic Information API FREE PARTIAL DATA IN USE (elections/divisions)

What it covers: Address-based lookup of elected officials at all levels. Enter an address, get back officials from federal down to local.

Data includes: Official names, office titles, party, contact info, photo URLs, social media links, websites.

Limits: 25,000 queries/day free.

CRITICAL CHANGE: The Representatives API was turned down in April 2025. The Elections and Divisions APIs continue to work. You can still look up elections and divisions by address, but direct representative lookups may be limited.

Gaps: Coverage of local officials is inconsistent. Some districts have full data, others are empty.

Documentation

Cicero API (Azavea) PAID PLANNED (local tier)

What it covers: The recommended replacement for Google Civic's turned-down Representatives lookup. Address → legislative-district matching + elected-official lookup at federal, state, and local levels, with contact info, social handles, and district geometry.

Why it matters here: Our address-to-rep feature leans on Google Civic, whose Representatives API was turned down (April 2025) — a migration is needed. Free bridge: Google's Divisions API now returns OCD-IDs for an address, which we can map to officials in our own + Congress.gov data (federal + state) without a paid vendor; Cicero (or BallotReady / Ballotpedia) covers the local tier we can't assemble for free.

Cost: Contact for pricing (tiered by lookups).

Cicero API

CivicEngine API PAID (pricing unclear)

What it covers: Election data and candidate info at all levels for jurisdictions with 5,000+ people. Federal, state, county, municipal, and school board races.

Features: Query by address or lat/lng, voting locations, election dates.

Cost: Contact required. No public free tier.

BallotReady API PAID

5. Map & Geographic Data

US Census Bureau TIGER/Line Shapefiles FREE / PUBLIC DOMAIN IN USE

What it covers: Official boundary data for congressional districts, state legislative districts, counties, cities, school districts, voting precincts, and more.

Formats: Shapefile, KML, GeoPackage. Can be converted to GeoJSON.

Latest: 2025 TIGER/Line files released September 2025 (boundaries as of January 1, 2025).

This is essential for: Your interactive map with district-level zoom.

TIGER/Line Downloads | Cartographic Boundary Files

Map Library Recommendation

Recommendation: Start with Leaflet, upgrade to MapLibre GL JS if needed.
  • Leaflet — Free, open-source, 38KB bundle. Great for basic interactive maps. No usage limits.
  • MapLibre GL JS — Free, open-source fork of Mapbox GL JS. GPU-accelerated, handles large datasets (1000+ features) at 60fps. Best default for new projects in 2026.
  • Mapbox — 50,000 free map loads/month, then paid. Beautiful tiles but vendor lock-in.

For your app with 400+ congressional districts plus state/local boundaries, MapLibre GL JS is the sweet spot — free, performant, and handles the data volume.

8. Recommended Strategy

Phase 1 Prototype: Build with Free Data

For the 2-3 month prototype, we can build a fully functional app covering federal and state officials using entirely free APIs. This gives you Congress members, state legislators, voting records, bills, campaign finance, and full interactive maps — which is already more than most civic apps offer.

Phase 2: Fill Local Gaps

For sheriffs, judges, DAs, and school board members, the options are:

  • Option A: Contact Ballotpedia for API pricing (likely the best comprehensive source).
  • Option B: Start with a single state (yours) and manually curate local data as proof-of-concept.
  • Option C: Build a community contribution system where local users can submit/verify data (long-term).

Key Decisions Needed

  • Do we start with federal + state data only, or attempt local from day one?
  • Which state should be the pilot for local-level testing?
  • Should criminal/education history be a later phase (since no API exists)?
